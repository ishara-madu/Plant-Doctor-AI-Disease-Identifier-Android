# Fix: Repeated analysis on back button press in Diagnosis Result page

## Issue
When user presses back button from Diagnosis Result page, the plant analysis runs again automatically.

## Root Cause
- In `MainActivity.kt` lines 396-409: The `LaunchedEffect(imageUriString)` runs analysis whenever:
  1. `imageUri` is present
  2. `diagnosisState is DiagnosisState.Idle`
- When back button is pressed, the `onBack` callback calls `diagnosisViewModel.resetState()` which sets diagnosisState back to Idle
- During navigation transition, the ResultScreen composable still exists and recomposes
- The LaunchedEffect fires again with existing imageUriString and new Idle state

## Solution
The fix will add a guard to prevent re-analysis after the image has already been processed once. We'll add a remember flag that tracks whether we've already started analysis for this image.

## Changes Required
Only one file needs modification:
**File**: `/Users/ishara/Documents/PlantDoctor/app/src/main/java/com/pixeleye/plantdoctor/MainActivity.kt`
**Lines**: Around line 396

Modify the fresh capture result composable to:
1. Add a `remember` flag to track if analysis has been triggered
2. Only run analysis if both conditions are met AND we haven't already run it
3. Set the flag to true after analysis starts

## Implementation Steps
1. Add `var analysisTriggered by remember { mutableStateOf(false) }` inside the composable
2. Modify the LaunchedEffect condition to also check `!analysisTriggered`
3. Set `analysisTriggered = true` immediately before starting analysis

This is a minimal fix that doesn't change any existing behavior except preventing the duplicate analysis run.
