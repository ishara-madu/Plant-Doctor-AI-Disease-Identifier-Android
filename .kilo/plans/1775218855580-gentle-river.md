# Fix: Add loading state and click prevention to Camera confirm button

## Issue
When user clicks on confirm button after taking a picture, there's around 1 second delay before any UI feedback (ad popup, analyzing screen). The button stays active during this time, allowing rapid multiple clicks which causes duplicate actions.

## Root Cause
1. `diagnosisViewModel.checkQuota()` performs synchronous database I/O on main thread inside the click handler
2. No loading state shown during quota check
3. No click prevention logic on confirm button
4. No debouncing or disabled state

## Solution
Add a loading state pattern similar to the existing CaptureButton implementation. This will:
1. Show a circular progress indicator inside the confirm button when processing
2. Disable button clicks during processing
3. Prevent rapid multiple clicks
4. Provide immediate visual feedback to the user

## Changes Required
Only one file needs modification:
**File**: `/Users/ishara/Documents/PlantDoctor/app/src/main/java/com/pixeleye/plantdoctor/ui/screens/CameraScreen.kt`

## Implementation Steps

### 1. Add loading state variable
Add a `var isConfirming by remember { mutableStateOf(false) }` in the CameraScreen composable where `capturedUri` is defined.

### 2. Modify ConfirmationContent to accept loading state
Update the `ConfirmationContent` function signature to add `isLoading: Boolean` parameter.

### 3. Update ActionButton to support loading state
Enhance the `ActionButton` component to:
- Accept `isLoading: Boolean` parameter
- When loading: replace icon with CircularProgressIndicator
- Disable clicks when loading
- Maintain existing appearance otherwise

### 4. Update confirm click handler
Wrap the existing logic with loading state:
```kotlin
onConfirm = {
    if (!isConfirming) {
        isConfirming = true
        scope.launch {
            try {
                // Existing quota check and navigation logic
            } finally {
                isConfirming = false
            }
        }
    }
}
```

### 5. Pass loading state to ConfirmationContent
When calling ConfirmationContent, pass `isLoading = isConfirming`.

## Implementation Notes

- This pattern exactly matches the existing `CaptureButton` implementation that already works correctly
- All required imports already exist (CircularProgressIndicator is already imported)
- No new dependencies needed
- The button will automatically reset its state if quota check fails or completes
- This prevents duplicate clicks during the database operation
- Provides immediate visual feedback that something is happening
